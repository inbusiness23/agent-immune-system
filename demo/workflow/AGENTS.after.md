# Agent Instructions

## Worktree Safety

Before editing, confirm the worktree is safe: correct branch, writable Git metadata, no unrelated dirty changes, and no active writer collision.

## Shell Safety

Proceed with safe read-only discovery. Ask before destructive commands, external writes, credential access, or irreversible actions.

## Visual Proof

Use browser or mobile proof when the user asks for visual evidence or when frontend changes need acceptance verification. Keep artifact ID conventions inside the proof-review workflow.

## Session Learning

Treat mistakes as evidence, not automatic permanent rules. Promote only repeated, damaging, generalizable lessons; otherwise route to watchlist, incident memory, reference, pruning, or regeneration.
