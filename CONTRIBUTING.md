# Contributing

Agent Immune System exists to reduce instruction bloat, not spread it.

## Contribution Bar

New rules, rubrics, examples, or hooks should be:

- specific
- verifiable
- minimal
- generalizable
- non-autoimmune
- safe by default

## Before Adding Guidance

Ask:

1. Is this issue damaging?
2. Is it recurring or critical?
3. Is the lesson generalizable?
4. Is it already covered?
5. Is the response proportional?
6. Could this make agents worse by overcorrecting?
7. Does it belong in active instructions, a skill, a reference, or dormant memory?

## Pull Requests

Good PRs include:

- the failure mode being addressed
- evidence or example
- the smallest sufficient change
- why this will not add prompt bloat
- before/after behavior when relevant

Avoid PRs that add broad "never do X" rules without recurrence evidence.
