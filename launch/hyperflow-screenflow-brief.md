# HyperFlow Screenflow Brief

Use this brief to render a chat-style screenflow from the examples in `examples/`.

## Goal

Create a 45-65 second screenflow showing a developer using Agent Immune System to audit bloated agent instructions.

## Source

- `assets/screenflow/chat-demo.json`
- `examples/01-postmortem-to-watchlist.md`
- `examples/02-bloated-agents-md-pruning.md`
- `examples/03-duplicate-rule-regeneration.md`
- `examples/04-skill-vs-global-routing.md`
- `examples/05-autoimmune-instruction-finding.md`

## Style

- Clean developer-tool UI.
- Dark editor panel plus light chat panel.
- High contrast text.
- No medical-heavy visuals.
- Use subtle immune metaphor language only: watchlist, antibody, autoimmunity, regeneration.
- Motion should feel like a product demo, not a cinematic trailer.

## Beat Map

1. Hook: user complains about bloated `AGENTS.md`.
2. Watchlist: one missed browser-proof event should not become a global rule.
3. Prune: old permission rules suppress safe work.
4. Regenerate: five git safety rules collapse into one durable principle.
5. Autoimmune: "never run commands" gets rewritten into a proportional rule.
6. Close: proposal-first, human approval before active edits.

## On-Screen Text

Primary title:

```text
Agent Immune System
```

Tagline:

```text
Detect prompt bloat. Prevent rule autoimmunity. Regenerate healthier agent instructions.
```

Decision badges:

```text
Watchlist
Prune
Regenerate
Autoimmune Finding
Human Approval Required
```

## Acceptance

- Viewer understands the skill from the screenflow without narration.
- Each example maps to one clear decision.
- Text is readable in a GitHub README embed.
- End frame includes repo name and tagline.
- Do not imply automatic rewrites.
