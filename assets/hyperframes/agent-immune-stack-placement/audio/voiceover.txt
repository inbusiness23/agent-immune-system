Your agent rules are making you slower.

Every time a session goes sideways, the reflex is to add one more rule to AGENTS.md.

That feels safer for about a day. Then the exceptions pile up, useful behavior gets blocked, and the agent starts carrying every old wound into every new task.

Agent Immune System changes the loop.

It treats mistakes as evidence, not automatic permanent memory.

In Codex, Claude Code, Cursor, terminal workflows, GSD, or GStack, the problem is the same: postmortems and review findings flow back into active instructions without a filter.

Agent Immune System sits on that instruction-health layer.

It asks: tolerate it, watch it, archive it, demote it, delete it, regenerate it, or only rarely promote it.

The result is not a bigger rules file.

The result is cleaner active instructions, dormant incident memory, and fewer autoimmune rules slowing the developer-agent loop.

In the included demo fixture, cleanup cuts the replay from eight minutes twenty to five minutes.

Not a universal guarantee. A proof artifact you can inspect, run, and improve.

Subtract the scar tissue before another postmortem becomes permanent.
